# TeamFinder

Платформа для поиска команды на pet-проекты. Реализована базовая функциональность из задания и вариант 2: навыки пользователей с добавлением без перезагрузки страницы и фильтрацией участников по навыкам.

## Структура

Проект приведён к структуре шаблона `team-finder-ad`:

- `team_finder/` — настройки Django-проекта и приложение с моделями, формами, views, тестами и management-командами;
- `templates_var2/` — используемые HTML-шаблоны для выбранного варианта 2;
- `templates_var1/`, `templates_var3/` — резервные папки под остальные варианты;
- `static/` — CSS и JavaScript;
- `media/` — пользовательские файлы;
- `docs/` — документация проекта.

## Запуск

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env_example .env
python manage.py migrate
python manage.py createsuperuser
python manage.py runserver --insecure
```

Сайт откроется по адресу `http://127.0.0.1:8000/`.

Для заполнения демо-данными:

```bash
python manage.py seed_demo
```

Логины: `anna@yandex.com`, `igor@yandex.com`, `maria@yandex.com`. Пароль для всех: `testpass123`.

## Запуск через Docker Compose

```bash
copy .env_example .env
docker compose up --build
```

В этом режиме используется PostgreSQL и volume `postgres_data`.

## Проверка

```bash
python manage.py test
```

CI также настроен через GitHub Actions: при push и pull request запускаются Django-тесты.

## Что есть в проекте

- регистрация и вход по email;
- публичные профили пользователей;
- редактирование профиля владельцем;
- создание, редактирование и завершение своих проектов;
- присоединение к открытым проектам;
- список пользователей и список проектов с пагинацией по 12 элементов;
- навыки в профиле с автодополнением и созданием новых навыков;
- фильтр пользователей по выбранному навыку;
- смена пароля для авторизованных пользователей;
- админ-панель для управления пользователями, профилями, проектами и навыками.

## Выполненный вариант

В проекте реализован вариант 2: навыки пользователей и фильтрация участников по навыкам.

## Чек-лист для ревьюера

- Проект запускается локально и через Docker Compose.
- В Docker Compose используется PostgreSQL и volume `postgres_data`.
- В корне есть `requirements.txt` со всеми зависимостями.
- Есть демо-пользователи, у каждого создаётся минимум один проект командой `seed_demo`.
- Добавлен CI через GitHub Actions.
- Есть тесты на основные страницы, создание проекта, права доступа, фильтр и навыки.
- Вариант 2 реализован: навыки отображаются в профиле, добавляются и удаляются без перезагрузки страницы, работает автодополнение и фильтрация `/users/list/?skill=<Название>`.
